Quarentine Cook

Created with HTML, CSS, PHP & MySQL

For this project, I created a recipe website. I used PHP and MySql to build a database of recipes and input them into my html website.

The Challenge

This website is a product of my term-long assignment for Scripting II. We were given a library of photos and recipes and instructed to create a "branded web application that would display the recipes and provide a user interface for browsing, searching and filtering through all of the available content." I spent four weeks developing the design and html structure of my site, then 5 weeks to build the database and make everything functional.

The Design Phase

My first steps in this project were to develop a style guide (including fonts, color schemes and imagery) and wireframe sketches.

The floral patterns in my design came from my physical recipe book, which can be found here. I thought it would be fun to connect the two worlds of recipes with this design.

The Development Phase

You can view my code for this project in my Github Repository. I also have a repository that was created before my MAMP files. This code isn't as up-to-date, but it shows my work developing the HTML & CSS.

The development of my site happened in two phases. The first phase was developing the design and structure. This is essentially the skeleton of my site and where the content from my database is input.

For my home page, I wanted to work with a simple tile structure. I added the small flower to spruce up the recipe titles, but otherwise kept it simple for easy reading. In the header, I initially wanted to group all of the navigation together and keep my filters in a dropdown menu, but this wasn't sustainable for all screen sizes (particularly mobile).

My main goal for the recipe page was to make both the instructions and ingredients visible at the same time. I find that most recipe websites make users do a lot of scrolling, so I put my two divs side-by-side. Initially I made them scroll individually, but this proved outdated and problematic in review.

...and now for my > final! <




My final design structure comes with a few improvements. I broke apart my nav bar and listed out my filters below the heading. This decresed problems in its responsiveness and gave the top of my home page some room to breath. I made improvements to the breakpoints of my responsive CSS to keep the recipe tile sizes consistent.

The recipe page got the most exciting improvements. Rather than making the ingredients and instructions scrollable, I made the ingredients sticky so that they would always be displayed as the user scrolls through the instructions. I also gave the flower - which functions as a home button - a fun, spinning animation

Most importantly, I built out my site's content and functionality. I created a database for the recipe content in MySQL. Then, using id references in my php code, I was able to pull each recipe into the tiles, categorize them by filter, build a functional search bar and link the recipe details to their own recipe page.
